The strongest hard evidence for surveyed models was existence evidence, not format evidence: Mini 5 Pro owners describing standalone SRT output, and a Mavic 4 Pro owner explicitly mentioning .SRT files on aircraft storage.

## High-demand consumer models

### DJI Mini 5 Pro

- **Release status:** In users' hands by early 2026. No public firmware version string captured.
- **Sample sources:** "My mini 5 pro produce an SRT file with very much(to much) information" — MavicPilots thread 155324. First-hand existence claim; no file attached.
- **Sample confidence:** Reported first-hand, not downloadable.
- **Field layout:** Not decodable. Best inference: continues Mini 3/4 Pro square-bracket lineage (treat as unconfirmed until real file lands).
- **Firmware variation:** Unknown. No public before/after pair.
- **Parser effort:** Moderate if real sample keeps square-bracket structure. Blocker is lack of sample, not engineering difficulty.
- **Regex sketch:** None — no qualifying public sample.

### DJI Air 3S

- **Release status:** Officially announced 15 October 2024.
- **Sample sources:** None found.
- **Sample confidence:** None.
- **Field layout:** Not decodable. Safest guess: HTML-style extended (Air 3 successor).
- **Parser effort:** Moderate, assuming Air 3S preserves Air 3-style multiline blocks.
- **Regex sketch:** None — no qualifying public sample.

### DJI Mavic 4 Pro

- **Release status:** Publicly introduced 13 May 2025.
- **Sample sources:** MavicPilots thread 155866 — user states aircraft storage contained .SRT and .LRF files. Mentioned across two Mavic 4 Pros and two firmware versions. No file attached.
- **Sample confidence:** Reported first-hand, not downloadable.
- **Field layout:** Not decodable. Could be Air 3S-style multiline family, but Mavic 4 Pro's tri-camera system and new infinity gimbal may produce extra tokens. Treat as **possibly novel** until a real file proves otherwise.
- **Firmware variation:** .SRT output confirmed across at least two post-launch firmware states; no file pair to compare token drift.
- **Parser effort:** Large — first high-demand model where large budget should be assumed unless sample proves otherwise.
- **Regex sketch:** None — no qualifying public sample.

### DJI Neo

- **Release status:** Shipping; downloads page shows release notes dated 6 November 2024.
- **Sample sources:**
  - PyPI `dji-telemetry` package describes itself as tested with DJI Neo 2 (different model) and shows a square-bracket-style example. Suggestive of family direction, but not a downloadable Neo sample.
  - Forum post advertising telemetry overlay tooling lists Neo among supported families, without a public downloadable file.
- **Sample confidence:** None for Neo itself.
- **Field layout:** Unknown. Neo's unusual control modes make it possible that subtitle generation depends on controller/app workflow rather than only aircraft family.
- **Parser effort:** Large — evidence base too weak and too cross-model to assume inheritance safely.
- **Regex sketch:** None — no qualifying public sample.

### DJI Avata 3

- **Release status:** Not corroborated in reviewed sources.
- **Sample sources:** None.
- **Recommended action:** Skip.

### DJI Mini 4K

- **Release status:** Current shipping product.
- **Sample sources:** None.
- **Critical note:** MavicPilots thread says Mini 2 and Mini 3 can embed telemetry in the video rather than create separate .srt sidecars, while Mini 4 Pro creates separate .srt. Because Mini 4K is clearly Mini 2/SE-derived, the **sidecar-SRT assumption may not hold** for this model. Real question is not token layout but whether there is an external SRT to parse at all.
- **Parser effort:** Do not code Mini 4K before confirming an external SRT exists.
- **Regex sketch:** None — no qualifying public sample.

## FPV and cinema models

### DJI FPV v2 and original DJI FPV

- **FPV v2 release:** No distinct second-generation DJI FPV aircraft corroborated. Official FPV result = DJI Digital FPV System page, not a new aircraft. "FPV v2" is too ambiguous to target.
- **Original DJI FPV:**
  - `JuanIrache/DJI_SRT_Parser` README says DJI FPV is among models it has been tested with.
  - First-hand MavicPilots post says DJI FPV/goggles telemetry can appear inconsistent — overlay tool sometimes asks for .srt, sometimes behaves as if telemetry is embedded in video.
  - `CallMarcus/dji-drone-metadata-embedder` README does not list original DJI FPV among formal supported lines.
- **Parser effort:** Large — open question is whether the right acquisition path is a clean sidecar .SRT, goggles-side metadata, or embedded subtitles.
- **Regex sketch:** None — no qualifying public sample.

- **Field layout:** Unknown. Cinema/RTK platform with very different workflow from DJI Fly line — do not assume same subtitle families without specimen.
- **Parser effort:** Large.
- **Regex sketch:** None — no qualifying public sample.
